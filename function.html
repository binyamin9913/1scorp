Functions

In the first chapter, we argued that language c is a procedural language, which means that the program can be broken down into functions - any function with a defined role and can be used by calling the function.
The advantage is the simplicity of reading the program and understanding it.
A function, then, is a piece of code with a unique name (like main) with a clear purpose, for example - a function that reads data from the keyboard, prints, sorts, etc.
Function Definition Template:
returned_type function_name (list_of_parameters)
returned_type - The type of value returned by the function. For example, the main function does not return a value and so its returned value type will be void.
function_name - The name of the function. A significant name must be given to indicate the role of the function. As the main function name is called main and unchangeable.
list_of_parameters - List of parameters (variables), separated by commas. The parameters are the values ​​that the function needs to perform in order to perform its function.
In case no parameters are needed we will leave the function brackets blank.
 definition:
Arguments - The variables placed when calling a function.
Parameters - are the variables that appear within the parenthesis of the function. These include the arguments the program sends.
When we send as a 2D parameter, the first dimension can be omitted but the second must appear. By passing a one-dimensional array as a parameter, its size can be omitted.
When writing the functions above the main function (main), no function statement is required. Conversely, if the function is written after the main function it must be declared (in order for the main function to recognize it) as we have defined a function definition template or without the parameter name.
For example:
int sum (int s);
: Or so
int sum (int);
In both cases the compiler will know that the function will be defined below.
In writing the function and defining it you can use different names for arguments, ie if you send a function a variable named b, then the name b must not appear after the function but other names can be used. This is because the value of the variable is the one copied to the function parameter, ie when the function is called copied One by one the values ​​of the arguments found in a command call to and initialize the function parameters.
When calling a function we must provide it with the necessary values ​​according to the different parameters of the function so that while we call the function these values ​​are copied to the different parameters and the values ​​of the parameters can be changed without affecting the values ​​we sent.
The function can be called an unlimited number of times.
When passing variables to a function as parameters, the values ​​of the variables do not change after the function is executed. However, since the state of the state is different (no copying is done as already mentioned), even calling a function does not copy an array argument to the function parameter and a change in the function in the array values ​​will leave the array values ​​with the change even after the function is completed.
